# Security Policy

## Supported Versions

Security fixes are best-effort for:

- the latest commit on `main`
- the latest published `0.1.x` release line

## Reporting a Vulnerability

Please do not open a public issue with exploit details.

Use GitHub's private vulnerability reporting flow if it is enabled for this repository. If private reporting is not available, open a minimal public issue requesting a private contact path without including sensitive details or reproduction steps.

When reporting, include:

- affected version or commit
- impact and attack surface
- reproduction steps or proof of concept
- any suggested mitigation

I will acknowledge reports as quickly as possible, work on a fix, and coordinate disclosure once a patch or mitigation is available.
